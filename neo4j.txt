2. Initial Implementation
   Created a basic Knowledge Graph for a fraud detection use case using Neo4j.

Entities (Nodes):

- Person
- Account
- Transaction
- Device
- Location

Relationships:

- Person → OWNS → Account
- Account → SENDS → Transaction
- Transaction → TO → Account
- Person → USES → Device
- Device → LOCATED_IN → Location

---

3. Extension of the Graph (Today’s Work)

Extended the existing graph to simulate more realistic and complex fraud scenarios:

- Added new users → Person (User2), Person (User3)

- Added new accounts → Account (A3), Account (A4)

- Added new transactions → Transaction (20000), Transaction (8000)

- Created new ownership links:
  Person → OWNS → Account (User2 → A3, User3 → A4)

- Extended transaction flows:
  Account (A3) → SENDS → Transaction → TO → Account (A1)
  Account (A4) → SENDS → Transaction → TO → Account (A2)

- Introduced shared device pattern (fraud case):
  Person (User1) → USES → Device (DeviceX)
  Person (User2) → USES → Device (DeviceX)
  Person (User3) → USES → Device (DeviceX)

→ This helps in identifying suspicious behavior where multiple users are linked to the same device.
